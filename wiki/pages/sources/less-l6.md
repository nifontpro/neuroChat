## Краткое содержание

Задание шестого занятия: реализовать простого агента — отдельную сущность с инкапсулированной логикой запроса и ответа, принимающего пользовательский запрос, отправляющего его в LLM и выводящего результат.

## Требования задания

- Принимает запрос пользователя
- Отправляет запрос в LLM через API (HTTP-клиент)
- Получает ответ
- Выводит результат в интерфейсе (чат, CLI или Web)
- Агент — отдельная сущность, не просто один вызов API
- Логика запроса и ответа инкапсулирована внутри агента

## Статус в NeuroChat

✅ **Реализовано**

| Требование | Где реализовано |
|---|---|
| Отдельная сущность-агент | `ChatViewModel` — принимает `ChatAction`, хранит состояние, оркестрирует вызовы |
| Инкапсулированная логика запроса | `OpenAiClient` — HTTP POST + SSE, `ChatRepository` — публичный фасад с маппингом ошибок |
| Приём запроса пользователя | `ChatAction.OnSendMessage` → `sendMessage()` → `startStreamingForUserMessage()` |
| Вывод результата | `ChatScreen` — Compose Multiplatform UI, токены стримятся в реальном времени |

Архитектура превосходит минимум задания: агент поддерживает историю контекста, стриминг токенов, обработку ошибок и команды.
