## Краткое содержание

Задание восьмого занятия: добавить подсчёт токенов для текущего запроса, всей истории и ответа модели; сравнить короткий/длинный/переполненный диалог; показать рост стоимости и что ломается при переполнении.

## Требования задания

- Подсчёт токенов для текущего запроса
- Подсчёт токенов для всей истории диалога
- Подсчёт токенов для ответа модели
- Сравнение: короткий / длинный / переполненный диалог
- Показать рост стоимости/токенов
- Показать что ломается при переполнении

## Статус в NeuroChat

✅ **Реализовано**

| Требование | Реализация |
|---|---|
| Токены текущего запроса | `lastUsage.promptTokens` — точное значение от провайдера из SSE финального чанка |
| Токены ответа модели | `lastUsage.completionTokens` |
| Всего за запрос | `lastUsage.totalTokens` |
| Накопленные токены сессии | `sessionTotalTokens` — сумма `totalTokens` всех запросов текущей сессии |
| Время генерации, скорость | `ResponseStatistics.durationMs`, `tokensPerSecond` — клиентский замер |
| UI отображения | `StatisticsSection` в SettingsPanel (включается свитчем «Показывать статистику») |
| Переполнение контекста | Ошибка провайдера → HTTP 400 → `DataError.BAD_REQUEST` → сообщение в UI |
