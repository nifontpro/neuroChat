# Wiki Schema

Это конфигурационный документ для LLM-агента, ведущего вики внутри `wiki/`.
Агент — единственный автор файлов в `wiki/pages/`, `wiki/index.md` и `wiki/log.md`.
Человек добавляет источники в `wiki/raw/` и задаёт вопросы.

## Структура каталогов

```
wiki/
├── SCHEMA.md          # этот файл; правила и процедуры (читает агент перед работой)
├── index.md           # каталог всех страниц вики (по категориям)
├── log.md             # хронологический журнал операций (append-only)
├── raw/               # неизменяемые источники (статьи, pdf, заметки, транскрипты)
│   └── assets/        # изображения и бинарные вложения к источникам
└── pages/             # сгенерированные агентом страницы вики
    ├── sources/       # одна страница на каждый ингест-источник (саммари + ссылки)
    ├── entities/      # страницы сущностей (люди, продукты, места, организации)
    ├── concepts/      # страницы концепций, тем, идей
    ├── analyses/      # ответы на запросы, сравнения, синтезы (filed back from Query)
    └── overview.md    # верхнеуровневая сводка / тезис вики
```

## Сущности и поля

Каждая страница в `pages/` начинается с YAML-frontmatter и далее markdown.

### Source (`pages/sources/<slug>.md`)
```yaml
---
type: source
title: "Точное название источника"
slug: tolkien-gateway-overview
source_path: raw/2026-05-03-tolkien-gateway.md
ingested_at: 2026-05-03
authors: [имя1, имя2]
url: https://...
tags: [tag1, tag2]
references: [entities/frodo, concepts/one-ring]   # на какие страницы влияет
---

## Краткое содержание
...

## Ключевые тезисы
- ...

## Цитаты
> ...

## Связи
- [[entities/frodo]] — добавлены детали о происхождении
- [[concepts/one-ring]] — уточнён тезис о его силе
```

### Entity (`pages/entities/<slug>.md`)
```yaml
---
type: entity
name: "Каноничное имя"
slug: frodo
aliases: [Фродо Бэггинс, Frodo Baggins]
category: person                # person | product | org | place | event | other
tags: [...]
sources: [sources/tolkien-gateway-overview, sources/silmarillion-ch3]
updated_at: 2026-05-03
---

## Кто/что это
...

## Ключевые факты
- факт ([[sources/...]])

## Связи
- [[entities/...]] — отношение
- [[concepts/...]] — связанная концепция

## Открытые вопросы / противоречия
- ...
```

### Concept (`pages/concepts/<slug>.md`)
```yaml
---
type: concept
name: "Каноничное название"
slug: one-ring
tags: [...]
sources: [...]
related_entities: [entities/frodo]
related_concepts: [concepts/...]
updated_at: 2026-05-03
---

## Определение
...

## Эволюция понимания
- 2026-05-01: первая интерпретация ([[sources/a]])
- 2026-05-03: уточнение/противоречие ([[sources/b]])

## Связи
...
```

### Analysis (`pages/analyses/<slug>.md`)
```yaml
---
type: analysis
title: "Вопрос или тема анализа"
slug: comparison-x-vs-y
created_at: 2026-05-03
question: "Исходный вопрос пользователя"
sources: [...]
references: [...]
---

## Ответ / синтез
...

## Использованные источники
- [[sources/...]]
```

## index.md — формат

```markdown
# Index

_Обновлён: 2026-05-03 · страниц: N · источников: M_

## Overview
- [[overview]] — текущий тезис вики

## Sources
- [[sources/<slug>]] — одна строка-описание (ingested_at)

## Entities
### Person
- [[entities/<slug>]] — одна строка
### Product
- ...

## Concepts
- [[concepts/<slug>]] — одна строка

## Analyses
- [[analyses/<slug>]] — одна строка (created_at)
```

Правила:
- Категории внутри `Entities` соответствуют `category` во frontmatter.
- Каждая строка ≤ ~120 символов; одна строка = одна страница.
- Сортировка алфавитная внутри категории; `Sources` и `Analyses` — по дате убыв.
- Любая правка `pages/**` обязана сопровождаться правкой `index.md`.

## log.md — формат

Append-only. Каждая запись — заголовок второго уровня с консистентным префиксом:

```markdown
## [2026-05-03 14:22] ingest | <Title>
- source: raw/<file>
- created: pages/sources/<slug>.md
- updated: pages/entities/a.md, pages/concepts/b.md (3 шт.)
- notes: что важно отметить (противоречие, новая сущность и т.п.)

## [2026-05-03 15:10] query | <короткий вопрос>
- answered_via: pages/sources/x, pages/entities/y
- filed_back: pages/analyses/<slug>.md   # если ответ оформлен как страница
- notes: ...

## [2026-05-03 16:00] lint
- contradictions: 2 (см. notes)
- orphans: 1 (pages/concepts/foo)
- stale: 0
- suggested_sources: ...
- notes: ...
```

Парсинг: `grep "^## \[" wiki/log.md | tail -N`.

## Slug-конвенция

- kebab-case, ASCII, без пробелов: `one-ring`, `tolkien-gateway-overview`.
- Источники: `<yyyy-mm-dd>-<short-title>` либо просто `<short-title>` если нет даты.
- Источник в `raw/` именуется тем же образом: `raw/2026-05-03-<short-title>.md`.
- Слаг уникален внутри своей категории.

## Кросс-ссылки

- Все ссылки между страницами — Obsidian-стиль `[[pages/entities/frodo]]` либо относительные `[[entities/frodo]]` (агент выбирает один стиль и держит его консистентно — по умолчанию относительный от `pages/`).
- Цитирование факта обязательно: `... ([[sources/<slug>]])`.
- При создании страницы добавляется в `references` источника и в `sources` целевой страницы.

# Процедуры

## INGEST

Вход: путь к новому файлу в `raw/` (md/pdf/txt/изображение).

Шаги (агент выполняет последовательно, отчитываясь по каждому):

1. **Прочитать источник целиком.** Для md/txt — Read. Для pdf — Read с pages. Для картинок — отдельным шагом после текста.
2. **Обсудить с пользователем ключевые тезисы** (3–7 буллетов) до записи. Дождаться подтверждения/правок.
3. **Создать `pages/sources/<slug>.md`** с frontmatter и саммари.
4. **Определить затронутые сущности и концепции:**
   - Для каждой упомянутой сущности — найти существующую страницу (через `index.md` + grep) или создать новую.
   - Для каждой ключевой концепции — то же.
5. **Обновить затронутые страницы:**
   - Добавить факты со ссылкой на источник.
   - Если новый факт противоречит существующему — отметить в секции "Открытые вопросы / противоречия" обеих сторон.
   - Обновить `updated_at`, `sources`, `references`.
6. **Обновить `index.md`** — добавить новые страницы, актуализировать счётчики.
7. **Добавить запись в `log.md`** по формату выше.
8. **Свериться с пользователем:** короткий итог — что создано, что обновлено, на что обратить внимание.

Бюджет правок: один ингест может затронуть 5–20 страниц. Если затрагивает >20 — остановиться и спросить.

## QUERY

Вход: вопрос пользователя.

Шаги:

1. **Прочитать `index.md`.** Выбрать кандидатов — страницы, чьи описания пересекаются с вопросом.
2. **При необходимости** `grep` по `pages/` для уточнения.
3. **Прочитать выбранные страницы** (обычно 3–10).
4. **Сформулировать ответ с цитатами** в формате `... ([[sources/<slug>]])`.
5. **Спросить, оформить ли ответ как `pages/analyses/<slug>.md`.** Если да — создать страницу, обновить `index.md` и затронутые сущности/концепции (добавить ссылку на анализ в их "Связи").
6. **Добавить запись в `log.md`** (тип `query`, с пометкой `filed_back` если страница создана).

## LINT

Запускается по запросу пользователя. Агент проходит по всей вики и пишет отчёт (обычно как `pages/analyses/lint-<date>.md` или просто в чат + запись в log).

Проверки:

1. **Противоречия:** факты на разных страницах со взаимоисключающими формулировками. Список с указанием страниц и источников.
2. **Устаревшие утверждения:** страница ссылается на источник, который позже опровергнут более новым источником. Использовать `ingested_at` источников.
3. **Orphans:** страницы в `pages/`, на которые нет ни одной входящей ссылки (grep по `[[<slug>]]`).
4. **Missing pages:** имена/термины, упоминаемые в ≥3 источниках, но без своей страницы.
5. **Broken refs:** ссылки `[[...]]` на несуществующие файлы.
6. **Index drift:** страницы в `pages/`, отсутствующие в `index.md`, и наоборот.
7. **Frontmatter gaps:** отсутствуют обязательные поля (`type`, `slug`, `updated_at`/`ingested_at`).

Отчёт — таблица "проблема → файл(ы) → предложенное действие".
Действия применяются только после подтверждения пользователем.

## Инварианты (агент обязан держать)

- `pages/**`, `index.md`, `log.md` — пишет только агент. `raw/**` и `SCHEMA.md` — правит человек.
- Любая запись факта на странице сопровождается ссылкой на источник.
- Каждая операция (ingest/query/lint) завершается записью в `log.md`.
- После любой правки `pages/**` `index.md` приведён в соответствие в той же сессии.
- Slug уникален; переименование = миграция всех ссылок.
- Не выдумывать факты. Если источник не подтверждает — не писать. Если неуверен — пометить `?` или вынести в "Открытые вопросы".
